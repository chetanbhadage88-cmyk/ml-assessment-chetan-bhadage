#question C-A single global model assumes homogeneous response across stores, which is unrealistic given differences in location, customer demographics, and competition. A hierarchical or segmented model captures both global patterns and local variations, improving prediction accuracy and enabling more tailored promotion strategies.

#B3 Model Evaluation and Deployment
#question A- A time-based train-test split should be used, where earlier data is used for training and the most recent months are reserved for testing, to reflect real-world forecasting conditions. A random split is inappropriate because it breaks the temporal order and can cause data leakage, leading to overly optimistic performance. Evaluation metrics should include RMSE to penalize large errors, MAE for average prediction error in units of items sold, and MAPE to assess relative accuracy across stores of different scales. These metrics together provide a comprehensive view of model performance and business impact.
#question B-In December, sales are already high due to seasonal demand, so the model recommends Loyalty Points to maximize customer lifetime value without heavy discounts. In March, demand is lower, so the model suggests a Flat Discount to attract more customers and increase sales volume.
#question C-At the start of each month, new data would be prepared by aggregating store-level features and combining them with calendar and promotion information, creating inputs for each store and promotion option. The saved pipeline would be applied to transform the data, and predictions would be generated to select the promotion with the highest expected sales for each store. For monitoring, I would track prediction errors (e.g., MAE, RMSE), detect data drift in input features, and monitor business KPIs such as actual sales uplift. Retraining would be triggered if model performance degrades or significant changes in data patterns are observed.
